---

## Context

- 현재 git 상태: !`git status`
- 스테이징되지 않은 변경 사항: !`git diff`
- 스테이징된 변경 사항: !`git diff --cached`
- 최근 커밋 이력: !`git log --oneline -10`

## 작업 절차

1. 위 컨텍스트를 바탕으로 모든 변경 파일을 파악한다.
2. 변경 파일을 아래 타입 기준으로 그룹화한다. 같은 타입이라도 도메인/관심사가 명확히 다르면 별도 커밋으로 분리한다.
3. 각 그룹별로 해당 파일만 `git add` 한 뒤 커밋을 생성한다.

## 커밋 타입 기준

| 타입       | 설명                                    |
| ---------- | --------------------------------------- |
| `feat`     | 새로운 기능 추가                        |
| `fix`      | 버그 수정                               |
| `refactor` | 기능 변경 없는 코드 구조 개선           |
| `style`    | 포맷, 공백 등 동작과 무관한 스타일 변경 |
| `test`     | 테스트 코드 추가/수정                   |
| `chore`    | 빌드 설정, 패키지, 설정 파일 등         |
| `docs`     | 문서 변경                               |
| `perf`     | 성능 개선                               |
| `build`    | 빌드 시스템 변경                        |

## 커밋 메시지 규칙

- 형식: `<타입>: <요약>`
- 반드시 **한국어**로 작성한다.
- 간결하고 명료하게 작성한다 (50자 이내 권장).
- `Co-Authored-By`, `Generated-By` 등 AI 귀속 트레일러는 **절대 추가하지 않는다**.

예시:

- `feat: 장바구니 수량 변경 기능 추가`
- `fix: 결제 페이지 총금액 계산 오류 수정`
- `refactor: ShippingForm 비즈니스 로직 훅으로 분리`
- `chore: ESLint 규칙 추가`
